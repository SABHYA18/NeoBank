NeoBank Super-App — Technical API Document

System Engine Release Baseline: 1.0.4-RELEASE

Global Context Base URL: https://api.neobank-superapp.local
Authorization Security Pattern: Standard JSON Web Token (JJWT Bearer Scheme attached via Authorization header)
Database Mutative Safeguard: Optimistic Version checking locks combined with explicit unique transactional tracking keys.

🔐 1. Authentication Layer (Public Endpoints)

POST/api/v1/auth/signup
Registers a brand new operational profile entity within the security identity relational datastore framework.

Example Request Payload

{
  "email": "customer@neobank.com",
  "username": "customer_neobank",
  "password": "Password123!",
  "fullName": "Jane Doe"
}
POST/api/v1/auth/login
Authenticates customer credentials and assigns short-lived access JWT arrays alongside durable refresh signature records.

Example Response (200 OK)

{
  "success": true,
  "message": "Authentication successful.",
  "data": {
    "accessToken": "eyJhbGciOi...",
    "refreshToken": "d7bca81d...",
    "username": "customer_neobank",
    "email": "customer@neobank.com",
    "role": "CUSTOMER"
  }
}

🏦 2. Account Management Layer (JWT Required)

POST/api/v1/accounts
Instantiates a new CHECKING or SAVINGS internal ledger accounting partition context initialized with an isolated zero base balance.

Example Request Payload

{
  "type": "CHECKING"
}
GET/api/v1/accounts
Retrieves arrays detailing all core functional balance instruments owned by the active validated caller session signature.

💳 3. Core Transactions & Double-Entry Ledger Layer

POST/api/v1/transactions/deposit
Executes an atomic balance top-up event. Strictly handles server retries using the tracking value supplied via X-Idempotency-Key header arrays.
POST/api/v1/transactions/transfer
Performs a reliable, atomic, double-entry bookkeeping balance adjustment moving funds between two valid target numbers.

Example Request Payload

{
  "fromAccountId": "acc-uuid-1234",
  "toAccountNumber": "0987654321",
  "amount": 2500.00,
  "description": "Rent payment",
  "idempotencyKey": "trans-idem-key-001"
}

💸 4. PayFlow Core Wallet Layer (JWT Required)

GET/api/v1/payflow/wallet
Fetches profile configurations for the authenticated wallet. Returns 404 Not Found if the wallet profile hasn't been instantiated yet.
POST/api/v1/payflow/wallet/initialize
Permanently binds a banking resource instrument UUID path and seeds the wallet with cash resources. Constraint rule: Minimum base deposit amount must equal or exceed ₹1,000.00.

Query Parameters: accountId (String), initialAmount (Decimal)

POST/api/v1/payflow/wallet/recharge
Pulls cash items straight from the mapped banking core interface account to bump current wallet resources.

Query Parameters: amount (Decimal)

POST/api/v1/payflow/send
Performs immediate cross-wallet user transfers. Validation rule: Request drops with an exception payload if sender wallet values drop below a structural ₹1,000.00 ceiling limit.

Example Request Payload

{
  "toUsername": "bobby_32",
  "amount": 400.00,
  "note": "Lunch share"
}
POST/api/v1/payflow/request
Saves an asymmetric pending accounting record requesting cash adjustments from another targeted username in the cluster.

Example Request Payload

{
  "fromUsername": "alice_w",
  "amount": 500.00,
  "note": "Movie ticket split"
}
POST/api/v1/payflow/requests/{id}/accept
Approves the incoming billing request item. Instantly debits current wallet resources (verifying the structural ₹1,000.00 floor remains safe) and credits the requester.
POST/api/v1/payflow/requests/{id}/decline
Cancels, drops, and marks a pending inbound billing tracking item payload as DECLINED.
