NeoBank Super-App API Contract Reference

System Topology Release: 1.0.4-RELEASE

Global Base URL: https://api.neobank-superapp.local
Authentication Scheme: JSON Web Token (JJWT Bearer Standard Validation Header)

🔐 1. Authentication Layer (Public)

POST/api/v1/auth/signup

Registers a brand new customer user entity within the persistent system security context mapping tables.

POST/api/v1/auth/login

Validates account credentials and issues signed runtime authentication tokens.

🏦 2. Account Management Layer (Authenticated)

POST/api/v1/accounts

Provisions a new core CHECKING or SAVINGS banking element linked directly to the client context profile.

GET/api/v1/accounts

Returns a summary list of all financial ledger balances held by the authorized user session.

💳 3. Core Transactions & System Ledger

POST/api/v1/transactions/deposit

Executes an atomic balance top-up event. Enforces idempotency via payload tracking tokens.

POST/api/v1/transactions/transfer

Performs a reliable, atomic, double-entry asset shift sequence across regional checking target mappings.

💸 4. PayFlow Polymorphic Wallet

POST/api/v1/payflow/pay

Dynamically routes third-party vendor requests (BILL, BOOKING, RECHARGE) through injected internal runtime strategies using a loose coupling factory pattern layout.
